# Web Share: Static Chat History Viewer & Export

## Overview

A static web page that renders Parley chat history as a polished, shareable showcase. Users can either preview conversations locally via drag-and-drop or export a self-contained HTML file for public hosting.

## Goals

- Produce a beautiful, read-only visualization of multi-agent conversations
- Enable public sharing via any static hosting (GitHub Pages, Netlify, S3, etc.)
- Single self-contained HTML file — no external dependencies, no build step
- Reuse rendering code between local viewer and exported files

## Architecture

Two modes, one template:

1. **Standalone viewer** (`web/share/template.html`) — Open in browser, drag-and-drop room files to preview locally. Shows empty state with drop zone when no data is loaded.

2. **CLI export** (`parley export <roomID>`) — Reads room files from `~/.parley/rooms/<roomID>/`, embeds the JSON data into the same HTML template, outputs a single `.html` file ready for hosting.

Both modes use identical CSS and JS rendering logic. The only difference is the data source: FileReader API (drag-drop) vs inline `<script>` tags with embedded JSON.

### Data flow

```
room folder (~/.parley/rooms/<id>/)
  ├── room.json        ─┐
  ├── messages.json     ─┼──▶  parley export  ──▶  single .html file (hostable)
  └── agents.json      ─┘
                                    │
                         same template.html
                                    │
                         viewer mode: drag-drop ──▶  same rendering
```

## Input format

Three JSON files from `~/.parley/rooms/<roomID>/`:

### room.json

```json
{
  "topic": "Refactor auth middleware",
  "id": "abc-123"
}
```

### agents.json

```json
[
  {
    "name": "Scott",
    "role": "human",
    "directory": "~/projects/myapp",
    "source": "human"
  },
  {
    "name": "Claude",
    "role": "code reviewer",
    "directory": "~/projects/myapp",
    "agent_type": "claude-code",
    "source": "agent"

  {
    "name": "Gemini",
    "role": "test writer",
    "directory": "~/projects/myapp",
    "agent_type": "gemini-cli",
    "source": "agent"
  }

```

### messages.json

```json

  {
    "id": "msg-1",
    "seq": 1,

  }

```

## UI design

### Visual style

Clean, modern, light theme. Similar to GitHub discussions or Linear — professional and approachable to non-technical viewers.

### Layout

Two-panel layout mirroring the TUI:

**Left sidebar (dark, ~220px):**
- Parley logo (SVG speech bubbles + wordmark + "multi-agent group chat" tagline)
- "Participants" section header
- Each participant showing:
  - Name (colored)
  - Agent type badge (e.g. "claude-code", "gemini-cli") — only for agents
  - Role as subtitle (e.g. "code reviewer", "test writer", "human")
  - Working directory

**Main chat area (light):**
- Topic bar at top with room topic + date + message count
- Message thread: avatar (first letter, gradient background) + name + timestamp + content
- Footer: "Generated by Parley — multi-agent group chat"

### Message rendering

- Avatar: circle with first letter of participant name, gradient background color
- Name: bold, followed by timestamp in muted text
- Content: 14px, 1.6 line height
- @mentions: highlighted in the mentioned participant's color
- Markdown: inline code, fenced code blocks, bold, italic, and line breaks. No tables, lists, or headings within messages.
- Participant colors: deterministic based on name hash, picking from a curated palette. Human participants get blue tones.

### Empty state (viewer mode)

When no data is loaded:
- Sidebar shows logo + "No data loaded"
- Main area shows a drag-and-drop zone:
  - Folder icon
  - "Drop a Parley room folder here"
  - Instructions showing expected files (room.json, messages.json, agents.json)
  - Path hint: `~/.parley/rooms/`
  - "Browse files" button as fallback

## Template design

Single HTML file containing all CSS and JS inlined. Key sections:

```html
<!DOCTYPE html>
<html>
<head>
  <style>/* all styles inlined */</style>
</head>
<body>
  <!-- Data injection point for export mode -->
  <script id="parley-data" type="application/json"></script>

  <div id="app"></div>

  <script>/* rendering logic + drag-drop handling */</script>
</body>
</html>
```

### Page load behavior

1. Check `#parley-data` for inline JSON content
2. If found (exported file) → parse and render immediately
3. If empty (standalone viewer) → show empty state with drag-and-drop

### Data injection (export mode)

The `#parley-data` script tag contains a JSON object:

```json
{
  "room": { ... },
  "agents": [ ... ],
  "messages": [ ... ]
}
```

## CLI command

```
parley export <roomID> [-o output.html]
```

1. Reads `room.json`, `messages.json`, `agents.json` from `~/.parley/rooms/<roomID>/`
2. Reads the HTML template (embedded in Go binary via `//go:embed`)
3. Injects the JSON data into the `#parley-data` script tag
4. Writes result to `<roomID>.html` (or custom path with `-o`)
5. Prints the output path to stdout

## Color assignment

Deterministic color from participant name:

1. Hash the participant name to an integer
2. If source is "human", pick from blue palette: `[#3b82f6, #2563eb, #1d4ed8]`
3. If source is "agent", pick from other colors: `[#8b5cf6, #059669, #d97706, #dc2626, #0891b2, #7c3aed]`
4. Each participant always gets the same color across renders

## Testing

- **Go unit test**: Given mock JSON files in a temp directory, call `Export()`, verify the output HTML contains the embedded data as valid JSON
- **Manual verification**: Open exported `.html` in browser, verify rendering. Open `template.html` standalone, verify drag-drop with real room files from `~/.parley/rooms/`
